## Mermaid Diagram Generation:

If you detect ANY of the above visual structures, create a Mermaid diagram representation in addition to the text transcription.

### Mermaid Syntax Guidelines:

**For Flowcharts:**
```mermaid
graph TD
    A[Start/Box Text] --> B{Decision?}
    B -->|Yes| C[Action 1]
    B -->|No| D[Action 2]
    C --> E((End))
    D --> E
```

**For Mind Maps:**
```mermaid
mindmap
  root((Central Idea))
    Branch 1
      Sub-point 1
      Sub-point 2
    Branch 2
      Sub-point 3
```

**For Sequence Diagrams (if showing interactions):**
```mermaid
sequenceDiagram
    participant A as Entity A
    participant B as Entity B
    A->>B: Action/Message
    B-->>A: Response
```

**For Entity Relationships:**
```mermaid
graph LR
    A[Entity 1] -->|relationship| B[Entity 2]
    B -->|relationship| C[Entity 3]
```

### Node Shapes to Use:
- `[Rectangle]` - Standard process/box
- `(Rounded)` - Rounded rectangle
- `((Circle))` - Circle/endpoint
- `{Diamond}` - Decision point
- `[(Database)]` - Database/storage
- `[/Parallelogram/]` - Input/output

### Arrow Types:
- `-->` - Solid arrow
- `-.->` - Dotted arrow
- `==>` - Thick arrow
- `--text-->` - Labeled arrow
